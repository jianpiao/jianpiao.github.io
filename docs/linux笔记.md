## cp：复制文件和目录

```basic
-a --archive 复制文件和目录，及其属性，包括所有权限
-i --interaction  覆盖前确认
-r --recursive  递归复制目录及内容
-u --update  复制到新的目录只更新新的目录中不存的内容
-v -- verbose 复制内容时显示信息
```



```bash
cp file1 file2 

# 覆盖前通知用户
cp -i file1 file2

# 将file1和file2复制到目录dir1里，dir1必须存在
cp file1 file2 dir1

#通配符将dir1中所有文件复制到dir2中
cp dir1/* dir2

# 将dir1目录复制到dir2目录，dir2目录不存在则创建
cp -r dir1 dir2
```

## chmod：权限

在平时我通过`ls -l`命令输出的结果，最前面10和字符会显示文件属性，现在就来仔细讲解一下。

命令拆分说明：`-rwx-r-r--`

### 文件类型(第一个字符)

```basic
-  普通文件
d  目录文件
l  符号连接。注意对于符号链接文件，剩下的文件属性始终是rwxrwxrwx，它是个伪属性值。符号链接指向的文件的属性才是真正的文件属性
c  字符设备文件
b  快设备文件，该文件类型表示以数据库方式处理数据的设备，如终端驱动或者光盘驱动
```

### 权限属性

```basic
r  允许打开和读取文件
w  允许写入或者截短文件，如果也设置了执行权限，那么目录中的文件允许，被创建被删除以及被重命名
x  允许把文件当做程序一样来执行，用脚本语言写的程序文件必须被设置为可读，以便能够被执行。
```

### 权限属性实例

```lua
-rwx------  普通文件，文件所有者具有读取、写入和执行权限、组成员和其他所有成员没有任何访问权限
-rw-------  普通文件，文件所有者具有读取、写入权限、组成员和其他所有成员没有任何访问权限
-rw-r--r-   普通文件，文件所有者具有读取、写入权限、文件所有者所在组的成员可以读取该文件，该文件对于所有用户都可读。
-rwxr-xr-x  普通文件，文件所有者具有读取、写入和执行权限，其他所有用户也可以读取和写入该文件。
-rw-rw----  普通文件，只有文件所有者和文件所有者所在的群组的成员具有读取和写入权限
lrwxrwxrwx  符号链接，所有的符号链接文件显示的都是”伪“权限属性，真正的权限属性由符号链接指向的实际文件决定
dwxrwx----  目录文件，文件所有者和所有者所在的去群组的成员都可以进入该目录，而且可以创建、重命名和删除该目录下的文件
drwxr-x---  目录文件，由上一条可知。
```

### 数字表示法

| 权限 | 权限数值 | 二进制   | 具体作用                                                     |
| ---- | -------- | -------- | ------------------------------------------------------------ |
| r    | 4        | 00000100 | **r**ead，读取。当前用户可以读取文件内容，当前用户可以浏览目录。 |
| w    | 3        | 00000010 | **w**rite，写入。当前用户可以新增或修改文件内容，当前用户可以删除、移动目录或目录内文件。 |
| x    | 1        | 00000001 | e**x**ecute，执行。当前用户可以执行文件，当前用户可以进入目录。 |

### 符号表示法

| 符号 | 含义                                 |
| ---- | ------------------------------------ |
| u    | user的简写，表示文件或者目录的所有者 |
| g    | 文件所属群组                         |
| o    | others的简写，表示其他所有用户       |
| a    | all的简写，是u，g，o三者的结合       |

### 设置权限

```bash
# 文件所有者，群，所有者都可以读写
chmod a+x test.txt

# 设置文件所有者具有可读、写入权限，群组和所有者没有任何权限，等同于 -rw------- 
chmod 600 test.txt

# 文件所有者，群，所有者都可以读写 等同于a+x
chmod 777 test.txt
```

### 环境是如何建立的

用户登录系统后，bash进程就会启动并读取一系列成为启动文件的配置脚本，这些脚本定义了所有用户的默认环境，接下来bash会读取更多存储在主目录下的用户定义个人环境的启动文件，这些步骤执行的确切顺序是由启动的shell回话类型决定的。

shell回话类型分为两种，分别为：login shell回话和non-login shell回话。

login shell回话会提示用户输入用户名和密码，如虚拟机控制台回话，而我们在GUI中启动的终端回话就是一个典型的non-logon shell回话

### login shell启动文件

| 文件            | 说明                                                         |
| --------------- | ------------------------------------------------------------ |
| /etc/profile    | 适用于所有用户的全局配置脚本                                 |
| ~/.bash_profile | 用户的个人启动文件，可扩展或重写全局配置脚本的设置           |
| ~/.bash_login   | 如果~/.bash_profile缺失，则bash尝试读取此文件                |
| ~/.profile      | 若~/.bash_profile与~/.bash_login均缺失，则bash尝试读取此文件 |

### non-login shell的启动文件

| 文件             | 内容                                                 |
| ---------------- | ---------------------------------------------------- |
| /etc/bash.bashrc | 适用于所有用户的全局配置脚本                         |
| ~/.bashrc        | 用户的个人启动文件，，可扩展或重写全局配置脚本的设置 |

## ssh：安全登录远程计算机

为了解决文明传送的问题，一个叫做SSH(Secure Shell)新协议应运而生，SSH协议解决了与远程主机进行安全通信的两个基本问题。

1. 该协议能验证远程主机的身份是否真实，从而避免中间人攻击
2. 该协议将本机与远程计算机之间的通信内容全部加密

SSH协议包括两个部分：

* 一个是运行在远程主机上的SSH服务端，用来监听端口22上可能过来的链接请求。
* 另一个是本机系统上的SSH客户端，用来与远程服务器进行通信。

> 1.硬链接，以文件副本的形式存在。但不占用实际空间。
>
> 2.不允许给目录创建硬链接
>
> 3.硬链接只有在同一个文件系统中才能创建

## at命令：

在用户指定的始课执行指定命令货命令序列
at [选项] [时间]
Ctrl + D  结束at命令的输入

### 选项：

-m 当指定的任务被完成后，将给用户发送邮件，即使没有标准输出
-c 打印任务内容到标准输出

### 时间定义：

格式hh:mm
模糊词：midnight，noon，teatime等
12小时制：12pm
指定具体日期：month day（月 日），mm/dd/yy或dd.mm.yy，yy-mm-dd,mmddyy
相对计时法：now+count time-units，now就是当前时间，time-units是时间单位，这里能够是minutes（分钟）、hours（小时）、days（天）、weeks（星期）。count是时间的数量，几天，几小时。例如：now + 5 minutes 04pm + 3 days
直接使用：today（今天）、tomorrow（明天）来指定完成命令的时间

## 相关命令：

atq：列出用户计划任务
atrm：根据jobnumber删除at任务
batch命令：在系统平均负载低于0.8时，立即执行成批命令货命令序列

### crontab命令

配置定时器，告诉提供定时器功能的cron守护进程让用户定时执行命令货命令序列
